# 全員一致ゲーム（COR） — 使い方とコマンド一覧

このファイルは `discordbot.py` に実装した COR（全員一致ゲーム）機能の使い方をまとめたものです。

## 概要
COR は次の流れで進行します。まず `!cor start` でお題を提示し、参加者は「このお題で続行するか」を `skip` / `continue` で決定します。`continue` が入力されると、開始チャンネルの参加者ごとに回答提出用のプライベートスレッド（作成不可時は DM）を用意します。参加者は自分用プライベートスレッドで回答文のみを送信すると提出扱いになります。最後に `!cor end` で提出状況を開示し、参加者が `yes` / `no` で一致判定を行います。

本実装では、以下を満たします:
- `!cor start` でゲーム開始（お題提示 + `skip` / `continue` 確認）
- `continue` 選択後、参加者ごとに回答提出用のプライベートスレッドを作成（失敗時は DM）
- 参加者は自分用のプライベートスレッドで回答文のみ送信して提出
- `!cor status` で提出済み / 未提出のみ表示（回答内容は表示しない）
- `!cor end` でゲーム終了、提出状況開示後に `yes` / `no` で成功/失敗判定

## 実行フロー（オペレーション）
1. 司会または任意のユーザーがチャンネルで `!cor start` を実行します。
   - ボットはお題を1つ提示し、同じチャンネルで `skip` または `continue` の入力を待ちます（120秒）。

2. `skip` が入力された場合。
   - お題を再抽選して再提示します。
   - 最大再抽選回数に到達した場合は開始をキャンセルします。

3. `continue` が入力された場合。
   - コマンドを実行したチャンネルの参加者（Bot除く）を参加者として確定します。
   - 各参加者に対して回答提出用のプライベートスレッドを作成し、提出方法を案内します。
   - スレッド作成に失敗したユーザーには DM で案内します。

4. 参加者は議論したうえで、自分用のプライベートスレッドに回答文をそのまま送信します。
  - 回答はスレッド所有者本人のものとして記録されます。
   - 同じ参加者が再提出した場合は最新の回答で上書きされます。

5. 進行中に `!cor status` を実行すると、提出状況を確認できます。
   - 参加者ごとに `提出済み` / `未提出` のみを表示します。
   - 回答本文は表示しません。

6. 司会が `!cor end` を実行するとゲーム終了です。
   - 提出状況（提出済み/未提出）を開示します。
   - その後、参加者（または開始者）が `yes` か `no` を入力して最終判定します。
   - `yes` で成功、`no` で失敗。時間内に入力がない場合は失敗として終了します。
   - ゲームで作成したプライベートスレッドは可能な範囲で削除し、状態をリセットします。

## コマンド詳細（例つき）
- `!cor start`
  - 説明: CORゲームを開始し、お題を提示して `skip` / `continue` の確認を行います。
  - 例: `!cor start`
  - 返信例:
```
全員一致ゲーム(COR)のお題: コンビニでつい買ってしまうものは？
このお題で良ければ continue、変更したければ skip と入力してください（120秒待機）。
```

- `skip` / `continue`（通常メッセージ）
  - 説明: `!cor start` 後のお題確認フェーズで使います。
  - 例: `skip`（お題を変更） / `continue`（このお題で開始）

- （プライベートスレッド内で）`<回答>`
  - 説明: 参加者が自分用スレッドで回答文のみ送信すると提出されます。
  - 例: `カレー`
  - 返信例: `提出を記録しました。`

- `!cor status`
  - 説明: 参加者全員の提出状況を表示します（回答内容は非表示）。
  - 例: `!cor status`
  - 返信例:
```
COR 現在のお題: コンビニでつい買ってしまうものは？
参加者数: 4
@A - 提出済み
@B - 未提出
@C - 提出済み
@D - 未提出
提出状況: 2/4
```

- `!cor end`
  - 説明: ゲームを終了し、提出状況を開示して最終判定（`yes` / `no`）を受け付けます。
  - 例: `!cor end`
  - 返信例:
```
COR ゲーム終了 - お題: コンビニでつい買ってしまうものは？
提出状況:
@A - 提出済み
@B - 提出済み
@C - 提出済み
@D - 提出済み
一致していると判定する場合は yes、失敗判定なら no を入力してください（120秒待機）。
```
  - 判定入力例: `yes`（成功） / `no`（失敗）

## 実装上の注意点 / 必要な権限
- プライベートスレッド作成/追加のため、サーバー設定に応じて以下の権限が必要です:
  - Create Private Threads
  - Send Messages / Send Messages in Threads
  - Manage Threads（環境によって必要）
- スレッド作成に失敗した場合は DM で提出案内を送ります。
- DM提出を受けるには、ユーザー側のDM受信設定が有効である必要があります。
- 現状のゲーム状態はメモリ内（プロセス内）管理です。ボット再起動で状態は失われます。

## 判定ルール（現状の実装）
- `!cor end` のあと、参加者（またはゲーム開始者）から `yes` / `no` のどちらか1件を受け付けます。
- `yes` が入力された場合: 成功（全員一致判定）
- `no` が入力された場合: 失敗
- 120秒以内に入力がない場合: 失敗として終了
